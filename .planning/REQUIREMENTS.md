# Requirements: SK Degree College Website

**Defined:** 2026-04-17
**Core Value:** To seamlessly connect all stakeholders—students, parents, and faculty—with the college's resources, information, and updates in one accessible, reliable place.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Core Architecture & Navigation

- [ ] **CORE-01**: Website is responsive and mobile-friendly on all standard devices.
- [ ] **CORE-02**: Main navigation includes links to Home, About, Academics, Admissions, Notice Board, and Contact.
- [ ] **CORE-03**: Users can search for content across the site using a global search bar.

### Informational Content (Public)

- [ ] **INFO-01**: Users can view the college's history, mission, and accreditation details.
- [ ] **INFO-02**: Users can view a list of all academic courses/programs offered.
- [ ] **INFO-03**: Users can view individual course details (duration, eligibility, syllabus overview).
- [ ] **INFO-04**: Users can view a faculty directory with profiles (photo, name, department, credentials).
- [ ] **INFO-05**: Users can view a photo gallery of campus events and facilities.

### Dynamic Notice Board

- [ ] **NEWS-01**: Users can view a chronological list of announcements/notices.
- [ ] **NEWS-02**: Notices include a title, date published, and detailed content.
- [ ] **NEWS-03**: Notices can be categorized (e.g., Exams, Admissions, Events).
- [ ] **NEWS-04**: Admin can create, edit, and delete notices via a headless CMS interface.

### Admissions & Inquiries

- [ ] **ADMN-01**: Prospective students can view admission criteria and fee structures.
- [ ] **ADMN-02**: Prospective students can submit an online inquiry/contact form.
- [ ] **ADMN-03**: Submitted inquiries are saved to the database and email notifications are sent to the admin.

### Student Portal (Authentication)

- [ ] **PORT-01**: Current students can log in using their credentials (email/password).
- [ ] **PORT-02**: Students can reset their password via an email link.
- [ ] **PORT-03**: Logged-in students can view a personalized dashboard.
- [ ] **PORT-04**: Logged-in students can view their basic profile information.

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Advanced Portal Features

- **PORT-05**: Students can view their internal exam grades/results.
- **PORT-06**: Students can pay fees online through a secure payment gateway.
- **PORT-07**: Faculty can log in and update their own profiles.

### Interactive Tools

- **TOOL-01**: Interactive campus map using Mapbox/Google Maps API.
- **TOOL-02**: AI-powered Chatbot for instant admission answers.

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Custom Learning Management System (LMS) | High complexity, not core to the initial value. We will use Moodle/Google Classroom instead. |
| Real-time messaging between students | Social features are outside the scope of an administrative/informational college portal. |
| Massive Video Backgrounds | Negatively impacts mobile performance and data caps for students. |
