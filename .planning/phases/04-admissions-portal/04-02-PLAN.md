# Plan: 04-02 - Secure Authentication Shell

## Goal
Implement the login and registration infrastructure for the student portal.

## Requirements
- PORT-01: Secure student login.
- PORT-02: Registration for new students.
- UX-03: Protected route redirection.

## Proposed Changes

### Components
- `components/LoginForm.tsx`: Login UI.
- `components/RegisterForm.tsx`: Registration UI.
- `components/AuthProvider.tsx`: Context provider for managing user state.

### Pages
- `app/login/page.tsx`: Entrance to the portal.
- `app/register/page.tsx`: Account creation.

### Middleware
- `middleware.ts`: Protect `/portal` and `/dashboard` routes.

## Verification Plan

### Automated Tests
- Build check: `npm run build`

### Manual Verification
1. Try to access `/dashboard` without logging in (should redirect to `/login`).
2. Login with dummy credentials and verify redirection to `/dashboard`.
3. Logout and verify access is revoked.
